# Magic26 Round 25｜UI 白話化 Copy Map

狀態：草案，僅規劃；尚未改 UI、未改資料 schema、未部署。

## 目標

把 Magic26 看板的使用者可見文字改成：

- 白話、短句、可掃讀。
- 保留交易/研究精度，不幼稚化。
- 主畫面不直接露出內部工程/研究黑話。
- 必要術語用短註解或 tooltip 保留原名。
- JSON schema、CSV 欄位、研究計算邏輯不改。

## 非目標

- 不重寫策略。
- 不改候選資格。
- 不改 A/B/C、Round 24 分組、排序、篩選的資料邏輯。
- 不刪資料下載。
- 不把研究看板改成交易訊號。

## 白話化原則

1. 主文案用中文白話；英文/縮寫只在必要處以括號或 tooltip 保留。
2. 內部欄位名保留在資料層，不直接露出在主畫面。
3. 「流程狀態」與「風險提示」分開呈現。
4. 高風險術語先標記為需確認，不硬翻。
5. 同一概念只能有一個主要 UI 名稱。
6. 使用者應能在不懂 Magic26 內部術語的情況下完成主要操作：看總覽、切分類、看候選、開詳情。

## 文案層級

- 主文案：白話名稱，短、直覺、可掃讀。
- 次文案：一句話說明用途或限制。
- tooltip / 小字：保留原術語、定義、研究口徑。
- 內部名稱：只留在 JS/JSON/CSV/schema，不作為可見 UI 主文案。

## 可見文字分類

- 頁面導覽：標題、副標、區塊名、按鈕。
- 資料時間：資料日期、更新時間、最新候選。
- 篩選條件：日期範圍、A/B/C、風險分類、排序。
- 候選狀態：優先程度、流程狀態、待確認、資料不足。
- 詳細欄位：入選原因、風險檢查、量能落差、後續表現、資料來源。
